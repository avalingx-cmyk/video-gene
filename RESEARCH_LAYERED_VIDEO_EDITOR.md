# Research: AI-Powered Video Editor (Canva-Style) — Correct Architecture

## User's Actual Vision

The user wants a **layered video creation platform** (like Canva) where each component is generated by a different AI, then composed together:

```
┌─────────────────────────────────────────┐
│  Layer 1: Video Footage                 │  ← AI Video Model (no text in prompt)
│  Layer 2: Text Overlays (brand, titles) │  ← AI Text Model (GPT/Claude) + User
│  Layer 3: Audio (voiceover + music)      │  ← AI TTS (Groq "qrog") + Music AI
│  Layer 4: Effects/Transitions           │  ← AI or Tool
└─────────────────────────────────────────┘
              ↓
         User Editor (like Canva)
         - Drag text positions
         - Adjust timing
         - Preview in browser
         - Export MP4
```

**Key insight**: Text is NEVER in the video generation prompt. Text is generated separately and overlaid on clean video footage. This avoids garbled text in AI-generated video.

---

## Why Previous Approach Failed

### What the user tried before:
- Put text/brand content in the video generation prompt
- AI video model tried to render text as pixels in the video
- Result: garbled, unreadable text, wrong brand colors, bad typography

### Why it failed:
- AI video models (Happy Horse, Luma, Runway) are trained on visual motion, not typography
- They cannot reliably render text, logos, or brand assets
- Text in prompts = unpredictable results (sometimes English, sometimes gibberish)

### The fix:
- **Generate video without text** — pure footage, motion, scenes
- **Generate text separately** — GPT/Claude for scripts, titles, brand copy
- **Overlay text programmatically** — use a tool/editor to place text on top of video
- **Result**: Clean, readable, brand-consistent text over professional video

## Recommended Architecture (Layered)

### Layer 1: Video Generation (Clean Footage, No Text)

**Primary API**: Happy Horse via fal.ai
- **Endpoint**: `alibaba/happy-horse/text-to-video`
- **Why**: #1 ranked, 1080p+audio, 9:16 native
- **Prompt strategy**: Describe scenes, motion, lighting, camera — NO TEXT
- **Example**: `"A man in a charcoal sweater looks out a window at sunset, soft diffused light, slow dolly forward, cinematic"`
- **Duration**: 5-15s per segment (3-5 segments for 30-75s video)
- **Cost**: $0.28/second at 1080p

**Fallback APIs**:
- Luma Ray2 (10s, 1080p, extendable)
- Runway Gen-4.5 (10s, highest quality)
- Kling 3 (15s, multi-shot)

### Layer 2: Text/Script Generation (Brand Content)

**Primary API**: GPT-4 / Claude
- **Why**: Best at marketing copy, educational scripts, brand voice
- **Input**: User prompt + content type (educational/marketing/tech)
- **Output**: 
  - Video script ( narration per segment)
  - Title overlays (per segment)
  - Brand slogans/taglines
  - Call-to-action text
- **Format**: JSON with timestamps and positions

**Example output**:
```json
{
  "segment_1": {
    "narration": "Welcome to our channel. Today we're learning about...",
    "title": "Introduction",
    "subtitle": "AI Video Creation",
    "cta": "Subscribe for more",
    "title_position": "top-center",
    "cta_position": "bottom-center"
  }
}
```

### Layer 3: Audio Generation (Voiceover + Music)

**Voiceover (TTS)**:
- **Primary**: Groq Orpheus ("qrog") — fastest, vocal directions like `[cheerful]`, `[whisper]`
- **Fallback**: ElevenLabs (32+ voices, 10K chars free/mo)
- **Budget option**: Fish Speech (open-source, 80+ languages, Apache 2.0)
- **Per-segment**: Generate TTS for each segment's narration

**Background Music**:
- **Option A**: Suno API (AI-generated music matching content mood)
- **Option B**: Royalty-free library (Epidemic Sound, Artlist — paid subscriptions)
- **Option C**: MiniMax Music v2.5/2.6 (fal.ai — text-to-audio with singing)
- **Auto-selection**: Based on content type (educational = calm, marketing = upbeat)

### Layer 4: Composition (Overlay + Mix)

**The critical question**: What tool overlays text, audio, and video?

**Option A: FFmpeg (Backend) + Canvas Editor (Frontend)**
- **Backend**: FFmpeg composes layers (video + text overlay + audio mix)
  - Text overlay: `drawtext` filter with custom fonts, colors, positions
  - Audio mix: `amix` filter for TTS + BGM with ducking
  - Transitions: `xfade` filter for crossfades
- **Frontend**: Canvas-based editor (like Canva) in React/Next.js
  - User drags text boxes, resizes, changes colors
  - Real-time preview with Remotion Player
  - Timeline scrubber for timing adjustments
- **Pros**: Full control, free (FFmpeg is open-source)
- **Cons**: Complex to build, text rendering quality depends on browser canvas

**Option B: Remotion (React-based Video Composition)**
- **What**: React components render to MP4 video
- **Text overlay**: React `<div>` with CSS styling renders as video frames
- **Audio**: Remotion handles audio tracks and mixing
- **Preview**: Remotion Player in browser (interactive)
- **Export**: Remotion Lambda or local render
- **Pros**: Perfect text rendering (browser-quality typography), React ecosystem
- **Cons**: $100/mo minimum for Remotion Automators license, complex setup

**Option C: Shotstack / Cloudinary (Video Editing APIs)**
- **What**: API-based video editing (send video + text + audio, get composed video)
- **Shotstack**: $49/mo, supports text overlays, transitions, audio mixing
- **Cloudinary**: Generous free tier, video transformations, text overlay
- **Pros**: No custom editor needed, handles composition automatically
- **Cons**: Paid services, less control than FFmpeg/Remotion

**Option D: Happy Horse Video Edit (AI Video Editing)**
- **Endpoint**: `alibaba/happy-horse/video-edit`
- **What**: Edit existing video with natural language instructions
- **Can do**: Recolor scenes, replace objects, change style, local/global edits
- **Cannot do**: Text overlay, brand logo placement, precise typography
- **Use case**: Post-processing video footage (change mood, season, style) but NOT text composition
- **Pricing**: $0.28/second at 1080p

**Recommendation**: 
- **For MVP**: FFmpeg (backend) + Canvas Editor (frontend)
  - Most control, zero cost, proven technology
  - Text rendered with browser canvas = high quality
  - Audio mixed server-side with FFmpeg
- **For Scale**: Remotion (if budget allows $100/mo)
  - Better developer experience
  - Professional text rendering
  - Built-in preview and export

## Complete Workflow

### Step 1: User Input
- User provides prompt: `"Create a 60-second educational video about AI video generation"`
- Optional: Upload brand logo, color palette, font preferences

### Step 2: AI Planning
- GPT-4 splits prompt into segments:
  - Segment 1 (0-15s): Introduction + hook
  - Segment 2 (15-30s): Main content
  - Segment 3 (30-45s): Deep dive
  - Segment 4 (45-60s): Call to action
- GPT-4 generates per-segment:
  - Video description (no text)
  - Narration script
  - Title text + position
  - Brand slogan

### Step 3: Parallel Generation
- **Video**: Happy Horse generates 4 segments (15s each) with clean footage
- **Audio**: Groq Orpheus generates 4 TTS clips from narration scripts
- **Music**: Suno/Minimax generates background music matching content mood
- **Status tracking**: Each job tracked independently via Celery + Redis

### Step 4: AI Composition (Draft)
- FFmpeg composes draft:
  - Concatenate 4 video segments with crossfades
  - Overlay text (titles, brand slogans, CTAs) at specified positions
  - Mix TTS audio + background music (auto-ducking)
- Output: Draft MP4 with all layers composed

### Step 5: User Review (Canva-Style Editor)
- Browser-based editor opens with draft video
- User can:
  - **Drag text** to new positions
  - **Change text** content, font, color, size
  - **Adjust timing** — when text appears/disappears
  - **Upload brand logo** and place it
  - **Change transitions** between segments
  - **Preview** in real-time
  - **Regenerate** a segment if footage is bad
- **Low-res preview**: 360p for fast iteration
- **Save state**: All edits stored as JSON (layer positions, timings, styles)

### Step 6: Final Export
- User clicks "Export"
- Backend re-composes with user's edits:
  - FFmpeg renders at 1080p with all overlays
  - High-quality audio mix
  - Fast-start MP4 for web playback
- **Output**: Final 1080x1920 MP4 ready for upload

### Step 7: Publishing (Optional)
- YouTube Data API: Upload + schedule publish
- n8n webhook: Trigger automation workflow
- Download: Direct MP4 download

## Research Agent Findings (Additional)

A research agent was deployed to investigate video composition tools. Key findings:

### Canva API: DOES NOT EXIST
- **Canva has no headless video API.**
- Canva only offers an Apps SDK for plugins running *inside* Canva.com.
- You cannot send JSON to Canva and get an MP4 back.
- **Implication**: We must build our own editor (not integrate Canva programmatically).

### Happy Horse Video-Edit Endpoint
- **Endpoint**: `alibaba/happy-horse/video-edit`
- **What it can do**: Recolor scenes, replace objects, change style, local/global visual edits
- **What it CANNOT do**: Text overlay, brand logo placement, precise typography
- **Implication**: Happy Horse video-edit is for post-processing footage, NOT for text composition.

### Shotstack API
- **Endpoint**: `POST https://api.shotstack.io/edit/v1/render`
- **What it does**: JSON timeline API — send video + text + audio, get composed MP4
- **Pricing**: ~$0.20–$0.30/minute of video
- **Text overlay**: Supported with styling
- **Audio mixing**: Supported
- **Pros**: Fastest to market, no custom editor needed
- **Cons**: Paid service, less control than FFmpeg/Remotion

### Remotion (React Video Composition)
- **What it does**: React components render to MP4 video
- **Preview**: `@remotion/player` — real-time browser preview
- **Export**: `@remotion/renderer` or Remotion Lambda
- **Pricing**: Free for individuals; $100/mo minimum for Automators; $500+/mo Enterprise
- **Why it wins**: The only tool where preview DOM/CSS is identical to final MP4 render. No "preview vs. final mismatch."

### FFmpeg (Free, Server-Side)
- **Text overlay**: `drawtext` filter with custom fonts, colors, positions
- **Audio ducking**: `sidechaincompress` filter for TTS over BGM
- **Transitions**: `xfade` filter for crossfades
- **Pros**: Free, full control, proven
- **Cons**: Complex to build, text rendering quality depends on server-side fonts

### Real-Time Preview Approaches
- **Remotion Player**: Frame-accurate browser scrubbing. Best option.
- **HTML5 Video + Canvas Overlay**: Works but preview ≠ final render (mismatch risk)
- **Fabric.js**: Good for drag-and-drop on canvas, best for editor UI

### Recommendations from Agent
1. **Best stack**: Remotion (React) for editor + preview + export
2. **Fastest to market**: Shotstack API for server-side composition + Canvas 2D preview
3. **Zero budget**: FFmpeg (backend) + Canvas 2D (frontend) — most control, most work

## Updated Conclusion

**The layered approach (video + text + audio generated separately, then composed in an editor) is the correct architecture.**

**We must build our own editor** because:
1. Canva has no API for programmatic video creation
2. We need full control over brand colors, fonts, logos
3. We need per-segment review and editing

### Recommended Tiered Approach:

| Phase | Tool | Cost | Purpose |
|-------|------|------|---------|
| **MVP** | Shotstack API | $0.20/min | Fastest to validate the idea |
| **V2** | FFmpeg + Canvas 2D | Free | Full control, custom editor |
| **V3** | Remotion + React | $100/mo | Professional preview+export |

**Next step**: If approved, build a Shotstack prototype first (1 week) to validate:
1. Generate 1 video segment (Happy Horse, no text)
2. Generate 1 title + script (GPT-4)
3. Generate 1 TTS clip (Groq "qrog")
4. Send to Shotstack (overlay text + mix audio)
5. Show result in browser
6. Let user adjust text position via Shotstack edit API
7. Export final MP4

This validates the architecture quickly before investing in custom editor development.
