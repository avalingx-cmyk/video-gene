# Project Plan: Video Generator - Segment-Based Workflow (Recommended)

## Executive Summary

Based on research, the **segment-based workflow** (5-15s clips stitched together) is the **only viable approach** for producing 30s-5min videos with current AI video APIs. No free API supports long videos in a single call.

**Key findings:**
- All major video APIs (Happy Horse, Luma, Runway, Kling) max at 10-15s per clip
- Quality degrades significantly beyond 10-15s
- Segment-based approach enables review, editing, and professional audio mixing
- Higher API cost (3-5x) justified by dramatically better UX and quality

---

## Improved Concept (Segment-Based)

**Video Generator** is a cloud-native video creation platform that produces publish-ready vertical videos (1080x1920) for YouTube Shorts, TikTok, and Instagram Reels using a **segment-based editing workflow**.

**How it works:**
1. User provides a text prompt (or .md/.pdf)
2. System splits prompt into 3-5 segment scripts (5-15s each)
3. Each segment is generated by AI video API (Happy Horse, Luma, Runway)
4. TTS audio is generated per segment (Groq Orpheus "qrog")
5. Background music auto-selected by content type
6. User reviews in browser timeline editor (Remotion Player)
7. User edits titles, transitions, reorder segments
8. FFmpeg composes final 1080p MP4
9. Optional: Upload to YouTube, trigger n8n workflow

**Core value proposition**: Zero-GPU video creation with review workflow. User sees preview before export, can edit segments, and gets professional audio mixing.

## Implementation Phases

### Phase 1: Segment Generation + Basic Editing
- FastAPI backend project structure
- Redis job queue (Celery) for async processing
- Input parser (text, .md, .pdf extraction)
- Content filter module (keyword-based)
- Prompt enhancer (educational/marketing/tech styles)
- Video router (Happy Horse, Luma, Runway, Kling providers)
- TTS service (Groq Orpheus "qrog", ElevenLabs, Fish Speech, Azure, Google)
- **Happy Horse via fal.ai integration** (15s segments, 1080p+audio, 9:16)
- Remotion setup (React + Next.js for video composition)
- Segment composer (stitch 3-5 segments into timeline)
- Basic title overlay per segment
- FFmpeg integration for MP4 output at 1080x1920
- Job status polling endpoint
- Segment model + API endpoint (`POST /api/v1/segments`)

### Phase 2: Transitions + Audio Mixing
- Transition effects (fade, wipe, slide via `@remotion/transitions`)
- Groq Orpheus TTS integration (fastest, vocal directions)
- ElevenLabs / Fish Speech TTS for voiceover
- Background music via royalty-free library
- Audio-video mixing (TTS + BGM + segments via FFmpeg)
- Crossfade transitions between audio segments
- Retry logic with exponential backoff + provider fallback chain

### Phase 3: Review Workflow + Export
- Preview player (Remotion Player in browser)
- Timeline scrubber UI (adjust segment timing, titles, reorder)
- Low-res preview generation (360p) before final export
- User adjustment interface (edit titles, change transitions)
- Final high-quality export (1080p MP4 via Remotion renderer or FFmpeg)
- YouTube Data API v3 integration (upload + scheduled publish)
- n8n webhook endpoints (POST /api/v1/generate, POST /api/v1/webhook/complete)

### Phase 4: Automation + Polish
- API key authentication for developer endpoints
- Per-user rate limiting (stay within API quotas)
- Cloud storage setup (S3/R2) for segments and final videos
- PostgreSQL metadata storage with per-user isolation
- User video library and prompt history UI
- Rate limiting and quota management dashboard
- Testing with real prompts across all content types
- API documentation for n8n and developer access
- Resolve open research items (copyright, misuse prevention, segment consistency)

## Cost Strategy (Zero Budget)

- **Segment generation**: Happy Horse via fal.ai (free credits on signup)
- **TTS voiceover**: Groq Orpheus (included with API access), ElevenLabs (10K chars/mo free), Fish Speech (open-source, self-hosted)
- **Video editing**: Remotion (free for individuals/small teams), FFmpeg (free, MIT)
- **Storage**: Cloudflare R2 (generous free egress)
- **YouTube API**: Free for uploads and scheduling
- **Review workflow**: Low-res previews (360p) to save on API costs
- **Fallback**: Fish Speech instead of ElevenLabs for budget-conscious users

## Security & Privacy

- User data never exposed externally
- API keys stored as environment variables
- Input content filtered before sending to third-party APIs
- Generated videos accessible only to creating user
- Per-user API key authentication for developer/n8n endpoints
- JWT token-based session management
- No watermarking (per requirement)
