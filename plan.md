# Project Plan: Video Generator — Segment-Based Layered Architecture (New Idea)

> **Status:** Plan mode — awaiting approval before implementation  
> **Date:** 2026-05-02  

---

## 1. Executive Summary

This plan replaces the old "full sequential video" approach with a **segment-based layered workflow**.

**The new idea:**
- Generate **short video segments** (5–15s) using an AI video model — **never put text in the video prompt**
- Generate **titles, text, and scripts** separately using an AI language model
- Generate **voiceover audio** separately using a TTS model (Groq Orpheus "qrog")
- Generate **background music** separately
- **Layer everything together** in an editing workflow: the user reviews and edits audio, title, animation, and transition before exporting the final video
- The result is a 30–100 second (or longer) professional video assembled from individually generated and independently editable layers

**Why this is better than the old plan:**
- AI video models **cannot render readable text** — putting "title" or "brand" in the prompt produces garbled, misspelled visual noise
- AI video models **degrade in quality after 10–15 seconds** — motion coherence collapses, characters drift, scenes break
- The old plan had **no review workflow** — one-shot generation with no editing, no fixing a bad segment
- The new plan gives the user a **Canva-like browser editor** to review, drag text, adjust timing, change transitions, and export only when satisfied

## 2. The Layered Workflow (The New Idea)

Each layer is generated by a different AI/tool, then composed in a browser editor:

```
┌─────────────────────────────────────────────────────────────┐
│  LAYER 1: AI Video Footage (NO text in prompt)            │
│  → Happy Horse / Luma / Runway / Kling                    │
│  → 5–15s segments, 1080×1920 (9:16), native audio          │
├─────────────────────────────────────────────────────────────┤
│  LAYER 2: Text / Titles / Brand Overlays                   │
│  → GPT-4 / Claude generates script, titles, CTAs           │
│  → User drags text on Canvas editor, changes font/color  │
├─────────────────────────────────────────────────────────────┤
│  LAYER 3: TTS Voiceover (per-segment narration)           │
│  → Groq Orpheus "qrog" — fastest, supports [cheerful] etc │
│  → ElevenLabs or Fish Speech as fallback                  │
├─────────────────────────────────────────────────────────────┤
│  LAYER 4: Background Music                                │
│  → Suno / MiniMax Music — auto-selected by content type   │
│  → Royalty-free library as fallback                       │
├─────────────────────────────────────────────────────────────┤
│  LAYER 5: Transitions (between segments)                   │
│  → Fade, wipe, slide via FFmpeg xfade                     │
│  → User selects per-segment transition in editor          │
└─────────────────────────────────────────────────────────────┘
                         ↓
         ┌──────────────────────────────┐
         │  Browser Canvas Editor         │
         │  - Drag text positions         │
         │  - Adjust timing (start/end)   │
         │  - Preview at 360p (fast)    │
         │  - Edit transitions           │
         │  - Approve → export           │
         └──────────────┬───────────────┘
                        ↓
         FFmpeg server-side render:
         - Concatenate segments + transitions
         - Burn text overlays at exact positions/times
         - Mix TTS + BGM with sidechain ducking
         - Output: 1080×1920 H.264 AAC MP4
```

### Critical Rule: Text NEVER Goes in the Video Prompt

**Wrong:** `"A cinematic video with the title 'Introduction' displayed at the top"`  
**Result:** Garbled, misspelled, unaligned text rendered by the video model.

**Right:** `"A cinematic shot of a person looking at a sunset, warm golden light, slow dolly forward, no text"`  
**Result:** Clean, professional footage. Title is added separately via Canvas overlay + FFmpeg `drawtext`.

## 3. Why This Is Better Than the Old Plan

| Aspect | Old Plan (Full Sequential Video) | New Plan (Segment-Based Layered) |
|---|---|---|
| **Video length** | Tried 30s–5min in one call (APIs fail at >15s) | 30s–5min via 3–20 segments of 5–15s |
| **Text quality** | Garbled, unreadable, wrong fonts | Crisp, brand-perfect typography via Canvas |
| **Brand control** | Random colors, random fonts | Exact brand colors, exact fonts, exact positions |
| **Editability** | None — one-shot generation | Full Canva-like editor: drag, resize, retime |
| **Review workflow** | Prompt → wait → video (hope it works) | Prompt → segments → edit → preview → export |
| **Audio sync** | Manual, error-prone guesswork | Precise per-segment TTS + FFmpeg ducking |
| **Fault tolerance** | One bad segment = entire video wasted | Regenerate only the bad segment |
| **Quality at 30s+** | Degrades dramatically | Each segment is optimal (5–15s = sweet spot) |

## 4. API Selection (2026 Research-Based)

### Video Generation (Layer 1: Clean Footage, No Text)

| Priority | API | Max Duration | Free Tier | 9:16 Native | Audio | Best For |
|---|---|---|---|---|---|---|
| **1 (Primary)** | **Happy Horse via fal.ai** | 15s | Credits on signup | ✅ Yes | Native 1080p+audio | Primary segment generation |
| **2 (Quality)** | **Luma Ray2** | 10s | Paid subscription | ✅ Yes | Native audio | Quality-sensitive segments |
| **3 (Top Elo)** | **Runway Gen-4.5** | 10s | Via Replicate | ✅ Yes | Native audio | Highest quality segments |
| **4 (Multi-shot)** | **Kling 3 via fal.ai** | 15s | Credits on signup | ✅ Yes | Multi-shot support | Character consistency |

**Key decision:** Start with Happy Horse via fal.ai (free credits, 15s, 1080p+audio). Fall back to Luma/Runway/Kling when Happy Horse fails or quota exhausted.

### Text Generation (Layer 2: Scripts, Titles, CTAs)

| Tool | Purpose | Cost |
|---|---|---|
| **GPT-4 / Claude** | Marketing copy, educational scripts, brand voice, per-segment titles | Included with API access |
| **Output format** | JSON with per-segment: narration, title, CTA, position, timing | — |

### TTS Audio (Layer 3: Voiceover)

| Priority | API | Languages | Vocal Directions | Cost | Best For |
|---|---|---|---|---|---|
| **1 (Primary)** | **Groq Orpheus "qrog"** | English + Arabic | ✅ [cheerful], [whisper], [sad] | Included with Groq API | Fastest per-segment TTS |
| **2 (Quality)** | **ElevenLabs** | 32+ voices | Style presets | 10K chars/mo free | Natural voice quality |
| **3 (Budget)** | **Fish Speech** | 80+ languages | Basic | $0 (open-source, self-hosted) | Budget-conscious users |

### Background Music (Layer 4: BGM)

| Option | Tool | Cost | Best For |
|---|---|---|---|
| **AI-generated** | **Suno API / MiniMax Music v2.5** | Free tier | Auto-matched mood, no copyright issues |
| **Licensed library** | Epidemic Sound, Artlist | Subscription | Professional, curated tracks |

### Video Composition (Layer 5 + Export)

| Tool | Role | Cost |
|---|---|---|
| **HTML5 Canvas 2D** | Browser preview/editor: text overlay, drag positioning, timing scrub | $0 |
| **FFmpeg** | Server-side export: segment concatenation, text burn-in, audio mixing, transitions | $0 |

**Why Canvas + FFmpeg instead of Remotion:**
- Remotion costs $100+/mo for commercial use (Company License)
- Canvas 2D gives identical real-time preview + drag-and-drop editing for $0
- Same JSON model drives both Canvas preview and FFmpeg export = zero WYSIWYG mismatch
- If budget becomes available later, migration to Remotion is straightforward

## 5. System Architecture

### 5.1 Data Flow

```
User Input (prompt / .md / .pdf / image)
         ↓
┌────────────────────────────────────────────┐
│  Backend: AI Planning Layer                │
│  - GPT-4 splits into 3–5 segment scripts  │
│  - Generates per-segment:                  │
│    • Video description (NO text)            │
│    • Narration script                      │
│    • Title + CTA text + position           │
│    • Transition type                       │
└───────────┬────────────────────────────────┘
            ↓ (parallel, tracked by Celery)
┌────────────────────────────────────────────┐
│  Backend: Generation Layer                 │
│  - Happy Horse: video segments            │
│  - Groq Orpheus: TTS per segment           │
│  - Suno: background music                  │
│  - Retry + provider fallback per job        │
└───────────┬────────────────────────────────┘
            ↓
┌────────────────────────────────────────────┐
│  Frontend: Canvas Editor (Browser)         │
│  - Layer panel: segments, text, audio      │
│  - Video preview at 360p (fast)             │
│  - Drag text, resize, change font/color    │
│  - Timeline scrubber (jump to any point)   │
│  - Transition selector per segment         │
│  - User approves → sends edit JSON to API  │
└───────────┬────────────────────────────────┘
            ↓
┌────────────────────────────────────────────┐
│  Backend: FFmpeg Composition               │
│  - Concatenate segments with xfade        │
│  - Burn text overlays (drawtext/ASS)      │
│  - Mix TTS + BGM (sidechain ducking)      │
│  - Output: 1080×1920 H.264 AAC MP4       │
└───────────┬────────────────────────────────┘
            ↓
    Final MP4 → Cloud Storage
            ↓
    YouTube API / n8n webhook / Download
```

### 5.2 Technology Stack

| Layer | Technology | Cost |
|---|---|---|
| Backend | Python 3.11 + FastAPI + Celery + Redis | $0 |
| AI Planning | GPT-4 / Claude (script splitting, title generation) | Included |
| Video Generation | Happy Horse via fal.ai (primary), Luma, Runway, Kling (fallback) | Free credits |
| TTS | Groq Orpheus "qrog" (primary), ElevenLabs, Fish Speech | $0–free tier |
| BGM | Suno / MiniMax Music (primary), royalty-free library | Free tier |
| Editor Frontend | React + Next.js + HTML5 Canvas 2D | $0 |
| Preview | HTML5 `<video>` + Canvas overlay synced to `currentTime` | $0 |
| Export | FFmpeg (concat, drawtext, xfade, sidechaincompress) | $0 |
| Storage | Cloudflare R2 | $0 |
| Database | PostgreSQL | $0 |
| Deployment | Railway / Render free tier | $0 |

## 7. Shared JSON Model (One Source of Truth)

The same JSON structure drives both the Canvas preview and the FFmpeg export. No mismatch.

```json
{
  "version": 1,
  "canvas": { "width": 1080, "height": 1920 },
  "segments": [
    {
      "id": "seg-1",
      "video_url": "https://cdn.example.com/segment1.mp4",
      "duration": 15,
      "transition": "fade"
    }
  ],
  "text_tracks": [
    {
      "id": "title-1",
      "text": "Introduction",
      "segment_id": "seg-1",
      "start": 2.0,
      "end": 12.0,
      "style": {
        "fontFamily": "Inter",
        "fontWeight": 700,
        "fontSize": 72,
        "color": "#ffffff",
        "outlineColor": "#000000",
        "outlineWidth": 4
      },
      "position": { "x": 540, "y": 200, "anchor": "center" }
    }
  ],
  "audio_tracks": [
    {
      "id": "tts-1",
      "type": "tts",
      "url": "https://cdn.example.com/tts1.mp3",
      "segment_id": "seg-1",
      "volume": 1.0
    },
    {
      "id": "bgm",
      "type": "music",
      "url": "https://cdn.example.com/music.mp3",
      "volume": 0.25,
      "ducking": true
    }
  ]
}
```

## 9. Risk Assessment

| Risk | Mitigation |
|---|---|
| Higher API cost (3–5× more calls) | Free tiers cover initial testing; Fish Speech replaces paid TTS; per-user rate limits |
| Segment inconsistency (character drift) | Reference images per segment (Happy Horse supports 1–9 refs); last-frame conditioning |
| Canvas preview ≠ FFmpeg export | Same JSON drives both; coordinate system identical (1080×1920) |
| Free credits exhausted | Provider fallback chain: Happy Horse → Luma → Runway → Kling |
| Audio sync issues | Per-segment TTS generated separately; FFmpeg concatenates with exact timestamps |
